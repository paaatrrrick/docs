## Commencez en trois étapes

Mettez votre site de documentation en marche localement et effectuez votre première personnalisation.

### Étape 1 : Configurez votre environnement local

<AccordionGroup>
  <Accordion icon="copy" title="Clonez votre documentation localement">
    Pendant le processus d'intégration, vous avez créé un dépôt GitHub avec le contenu de votre documentation si vous n'en aviez pas déjà un. Vous pouvez trouver un lien vers ce dépôt dans votre [tableau de bord](https://dashboard.mintlify.com).
    
    Pour cloner le dépôt localement afin de pouvoir apporter et prévisualiser des modifications à votre documentation, suivez le guide [Cloner un dépôt](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository) dans la documentation GitHub.
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Démarrez le serveur de prévisualisation">
    1. Installez la CLI Mintlify : `npm i -g mint`
    2. Naviguez vers votre répertoire de documentation et exécutez : `mint dev`
    3. Ouvrez `http://localhost:3000` pour voir votre documentation en direct !
    
    <Tip>Votre prévisualisation se met à jour automatiquement lorsque vous modifiez des fichiers.</Tip>
    
    <Check>Vous devriez voir la page d'accueil de votre documentation avec le style Mintlify par défaut.</Check>
  </Accordion>
</AccordionGroup>

### Étape 2 : Déployez vos modifications

<AccordionGroup>
  <Accordion icon="github" title="Installez notre application GitHub">
    Installez l'application GitHub Mintlify depuis votre [tableau de bord](https://dashboard.mintlify.com/settings/organization/github-app).
    
    Notre application GitHub déploie automatiquement vos modifications sur votre site de documentation, vous n'avez donc pas besoin de gérer les déploiements vous-même.
</Accordion>
<Accordion icon="palette" title="Mettez à jour le nom et les couleurs de votre site">
    Pour une première modification, mettons à jour le nom et les couleurs de votre site de documentation.

    1. Ouvrez `docs.json` dans votre éditeur.
    2. Changez le champ `"name"` pour le nom de votre projet.
    3. Mettez à jour les `"colors"` pour correspondre à votre marque.
    4. Enregistrez et voyez vos modifications instantanément à `http://localhost:3000`.

    <Tip>Essayez de changer la couleur primaire pour voir une différence immédiate !</Tip>
  </Accordion>
</AccordionGroup>

### Étape 3 : Mettez en ligne

<Accordion icon="rocket" title="Publiez votre documentation">
  1. Validez et poussez vos modifications.
  2. Votre documentation sera mise à jour et en ligne en quelques instants !
</Accordion>
