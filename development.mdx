<Info>
  **Prérequis** :
  - Node.js version 20 ou supérieure (LTS recommandé)
  - Un dépôt de documentation avec un fichier `docs.json`
</Info>

Suivez ces étapes pour installer et exécuter Mintlify sur votre système d'exploitation.

<Steps>
<Step title="Installez la CLI Mintlify">

```bash
npm i -g mint
```
</Step>

<Step title="Prévisualisez localement">

Naviguez vers votre répertoire de documentation où se trouve votre fichier `docs.json`, et exécutez la commande suivante :

```bash
mint dev
```

Une prévisualisation locale de votre documentation sera disponible à `http://localhost:3000`.

</Step>
</Steps>

## Ports personnalisés

Par défaut, Mintlify utilise le port 3000. Vous pouvez personnaliser le port sur lequel Mintlify s'exécute en utilisant l'option `--port`. Par exemple, pour exécuter Mintlify sur le port 3333, utilisez cette commande :

```bash
mint dev --port 3333
```

Si vous tentez d'exécuter Mintlify sur un port déjà utilisé, il utilisera le prochain port disponible :

```md
Le port 3000 est déjà utilisé. Essai du port 3001 à la place.
```

## Versions de Mintlify

Veuillez noter que chaque version de la CLI est associée à une version spécifique de Mintlify. Si votre prévisualisation locale ne correspond pas à la version de production, veuillez mettre à jour la CLI :

```bash
npm update -g mint
```

## Validation des liens

La CLI peut vous aider à valider les liens dans votre documentation. Pour identifier les liens cassés, utilisez la commande suivante :

```bash
mint broken-links
```

## Formatage du code

Nous suggérons d'utiliser des extensions sur votre IDE pour reconnaître et formater le MDX. Si vous utilisez VSCode, considérez l'[extension MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique, et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

## Dépannage

<AccordionGroup>
  <Accordion title='Erreur : Impossible de charger le module "sharp" en utilisant le runtime darwin-arm64'>

    Cela peut être dû à une version obsolète de node. Essayez ce qui suit :
    1. Supprimez la version actuellement installée de la CLI : `npm remove -g mint`
    2. Mettez à niveau vers Node v19 ou supérieur.
    3. Réinstallez la CLI : `npm i -g mint`
  </Accordion>

  <Accordion title="Problème : Rencontre d'une erreur inconnue">
  
    Solution : Allez à la racine de votre appareil et supprimez le dossier `~/.mintlify`. Ensuite, exécutez à nouveau `mint dev`.
  </Accordion>
</AccordionGroup>

Curieux de savoir ce qui a changé dans la dernière version de la CLI ? Consultez le [journal des modifications de la CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
