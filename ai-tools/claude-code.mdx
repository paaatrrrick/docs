Claude Code est l'outil CLI officiel d'Anthropic. Ce guide vous aidera à configurer Claude Code pour vous aider à rédiger et maintenir votre documentation.

## Prérequis

- Abonnement Claude actif (Pro, Max ou accès API)

## Configuration

1. Installez Claude Code globalement :

   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

2. Naviguez vers votre répertoire de documentation.
3. (Optionnel) Ajoutez le fichier `CLAUDE.md` ci-dessous à votre projet.
4. Exécutez `claude` pour démarrer.

## Créer `CLAUDE.md`

Créez un fichier `CLAUDE.md` à la racine de votre dépôt de documentation pour entraîner Claude Code sur vos normes de documentation spécifiques :

````markdown
# Documentation Mintlify

## Relation de travail
- Vous pouvez contester les idées - cela peut conduire à une meilleure documentation. Citez vos sources et expliquez votre raisonnement lorsque vous le faites
- Demandez TOUJOURS des clarifications plutôt que de faire des suppositions
- Ne mentez JAMAIS, ne devinez pas et n'inventez pas d'informations

## Contexte du projet
- Format : fichiers MDX avec frontmatter YAML
- Configuration : docs.json pour la navigation, le thème, les paramètres
- Composants : composants Mintlify

## Stratégie de contenu
- Documentez juste assez pour le succès de l'utilisateur - ni trop, ni trop peu
- Priorisez l'exactitude et l'utilisabilité des informations
- Rendez le contenu pérenne lorsque possible
- Recherchez les informations existantes avant d'ajouter du nouveau contenu. Évitez la duplication sauf si elle est faite pour une raison stratégique
- Vérifiez les modèles existants pour la cohérence
- Commencez par faire les plus petits changements raisonnables

## Exigences de frontmatter pour les pages
- title : Titre de page clair et descriptif
- description : Résumé concis pour le SEO/navigation

## Normes de rédaction
- Voix à la deuxième personne ("vous")
- Prérequis au début du contenu procédural
- Testez tous les exemples de code avant publication
- Correspondez au style et formatage des pages existantes
- Incluez des cas d'utilisation basiques et avancés
- Balises de langage sur tous les blocs de code
- Texte alt sur toutes les images
- Chemins relatifs pour les liens internes

## Flux de travail Git
- N'utilisez JAMAIS --no-verify lors des commits
- Demandez comment gérer les modifications non commitées avant de commencer
- Créez une nouvelle branche quand aucune branche claire n'existe pour les modifications
- Commitez fréquemment tout au long du développement
- Ne sautez ou désactivez JAMAIS les hooks pre-commit

## À ne pas faire
- Sauter le frontmatter sur tout fichier MDX
- Utiliser des URLs absolues pour les liens internes
- Inclure des exemples de code non testés
- Faire des suppositions - demandez toujours des clarifications
````
